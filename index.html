Intro to Programing notes

Lesson 1

Thinking like a programmer

Thinking like a programmer involves the combination of procedural thinking, abstract thinking,and systems thinking.

Procedural thinking involves creating perfectly clear and unambiguous instructions for a computer to follow.

Abstract thinking means finding similarity, or as programmers would say, generality amongst seemingly different things.

Systems thinking happens when you break a big problem down into smaller pieces. Programmers do this when they create a plan (often on paper) for how a program will work. It involves big-picture thinking and decision-making about a problem and how different pieces of a program can work together to solve it.

The World Wide Web

The web is made up of tons of computers that communicate with each other via HTTP(Hyper Text Transfer protocol). The major pieces of the web are you, and your computer, the internet itself, and servers serving .html pages.

HTML

HTML stands for Hyper Text Markup Language, it is made up of text and is the main type of document found on the web.

HTML documents are made of tags and elements, HTML tags tell browsers the type of elements via HTML. You can find more on html tags and elements.here

Inline and Block elements

Inline elements are a displayed without line breaks.

Block elements form an invisible box around the content.

Lesson 2

Why start with HTML?

HTML is very visual. As I make changes in codepen I can see my work take form, helping me catch errors or things that did not work, as I expected them to.

HTML is all about the boxes

Every element of a web page can be broken up into a series of boxes, using div tags and css.

A great analogy for HTML, and CSS

HTML is like the structure of a house. Where are the walls going to be? What room is the bedroom?. How high is the roof?

CSS is like the styling of the house. What color are those walls? Do I want tile floors or Carpet? What color will the tile/carpet be?

So to sum it up.. HTML is the structure, and CSS is the styling of the contents of the structure.

Lesson 3

All about CSS

CSS stands for Cascading Style Sheets. We can use CSS to avoid repeating ourselves, using the same code to apply styles to like portions of our HTML. Avoiding repetition helps us write cleaner and more efficient code. Less lines of code uses less bandwidth and is easier to troubleshoot.

Inheritance

Inheritance is the process by which properties are passed from parent to child elements even though those properties have not been explicitly defined by other means. Certain properties are inherited automatically, and as the name implies, a child element will take on the characteristics of its parent with regards to these properties.

Code, Test, Refine
  1. Look for natural boxes (everything on the web is a box)
  2. Look for repeated styles and semantic elements
  3. Write your html
  4. Apply Styles
  5. Fix things
Back to the house analogy

Build the house before you furnish it, and paint it. Build your html structure before you add contents and style, then come back with the paint/contents and style. This will make things flow better.
